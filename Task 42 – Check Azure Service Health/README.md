# Task 42 – Check Azure Service Health

## 🎯 Objective

Monitor Azure platform health and verify if any regional outages, service issues, or planned maintenance events are affecting resources.

---

## 📌 Overview

Azure Service Health provides information about:

- Service Issues (outages)
- Planned Maintenance
- Health Advisories

It helps determine whether issues are caused by Azure infrastructure rather than user configuration.

---

## 🛠️ Steps Performed

### Step 1: Navigate to Service Health
- Login to Azure Portal
- Search for **Service Health**
- Click to open

---

### Step 2: Review Health Information

Observed the following sections:

- **Service Issues** → Active Azure outages (if any)
- **Planned Maintenance** → Scheduled updates
- **Health Advisories** → Informational notices
- **Security Advisories** → Important security updates

---

### Step 3: Filter by Subscription

- Selected current subscription
- Verified health status specific to subscription

---

## 🔍 Observations

- Checked for active service issues.
- Reviewed planned maintenance notifications.
- Verified no regional outages affecting current resources.

---

## 📊 Outcome

- Successfully accessed Azure Service Health.
- Confirmed platform health status.
- Understood difference between platform-level and resource-level issues.
